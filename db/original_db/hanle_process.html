🧹 Ingredient Cleanup Workflow

Quy trình xóa Dish không đủ nguyên liệu sau khi cập nhật Ingredient DB

🤖
Bước A Feed AI danh sách Ingredient hợp lệ

Cho AI "ăn" toàn bộ danh sách ingredient_id + name từ DB hiện tại — đây là nguồn sự thật duy nhất.

SELECT ingredient_id, name FROM ingredient ORDER BY name; -- Kết quả: ~N dòng, lưu vào context AI
OUTPUT AI đã biết toàn bộ nguyên liệu hợp lệ: { uuid-ing-01: "bánh phở", uuid-ing-02: "thịt bò", ... }
📋
Bước B Query toàn bộ Dish + nguyên liệu thô

Lấy tất cả Dish kèm danh sách nguyên liệu hiện đang gắn với chúng trong bảng dish_ingredient. Lưu thành JSON.

SELECT d.dish_id, d.name, json_agg(json_build_object( 'ingredient_id', di.ingredient_id, 'name_raw', i.name )) AS ingredients FROM dish d JOIN dish_ingredient di USING(dish_id) JOIN ingredient i USING(ingredient_id) GROUP BY d.dish_id, d.name;
OUTPUT dishes.json — mảng { dish_id, name, ingredients[] }
📝
Bước C Tạo file TXT — batch input cho AI

Chuyển JSON thành định dạng text có cấu trúc để AI xử lý hàng loạt. Mỗi Dish là một block, phân cách bởi ---.

-- Cấu trúc mỗi block trong file .txt -- DISH_ID: 550e8400-e29b-41d4-a716 NAME: Phở bò INGREDIENTS: - bánh phở - thịt bò tái - hành tím - gừng nướng - nước mắm --- DISH_ID: ... NAME: ...
OUTPUT dishes_to_verify.txt — toàn bộ Dish cần AI kiểm tra
🧠
Bước D AI map nguyên liệu thô → ingredient_id

AI duyệt từng nguyên liệu trong mỗi Dish, so khớp với danh sách đã biết (Bước A), trả về ingredient_id hoặc NOT_FOUND.

-- Prompt gửi cho AI -- Dựa vào danh sách ingredient đã cung cấp, duyệt qua từng nguyên liệu trong dish sau. Nếu tìm thấy nguyên liệu tương ứng → trả về ingredient_id. Nếu không tìm thấy → trả về "NOT_FOUND". Trả về JSON duy nhất, không giải thích thêm: { "dish_id": "...", "mapped": { "bánh phở": "uuid-ing-01", "thịt bò tái": "uuid-ing-02", "gừng nướng": "NOT_FOUND" } }
OUTPUT mapping_result.json — mỗi dish có danh sách nguyên liệu đã được map
❓
Dish có nguyên liệu NOT_FOUND không?
✗ CÓ — incomplete dish
Ít nhất 1 nguyên liệu không map được → Dish không đủ điều kiện tồn tại trong DB → tiến hành xóa
✓ KHÔNG — dish hợp lệ
Tất cả nguyên liệu đều map được → Dish giữ nguyên, cập nhật FK nếu cần
🔍
Bước E Tổng hợp danh sách Dish cần xóa

Quét mapping_result.json, thu thập tất cả dish_id có ít nhất một giá trị NOT_FOUND.

-- Script Python / JS xử lý mapping_result.json -- dishes_to_delete = [ entry["dish_id"] for entry in mapping_result if "NOT_FOUND" in entry["mapped"].values() ] # Log để review trước khi xóa print(f"Sẽ xóa {len(dishes_to_delete)} dish")
OUTPUT delete_list.json — danh sách dish_id cần xóa (review trước khi chạy)
🗑️
Bước F Xóa cascade — dish_ingredient → dish

Xóa theo đúng thứ tự FK: xóa bảng junction trước, sau đó xóa Dish. Bọc trong transaction để rollback nếu có lỗi.

BEGIN; -- Bước 1: xóa junction table trước DELETE FROM dish_ingredient WHERE dish_id = ANY(:delete_list); -- Bước 2: xóa dish DELETE FROM dish WHERE dish_id = ANY(:delete_list); -- Kiểm tra trước khi commit COMMIT; -- hoặc ROLLBACK nếu cần
OUTPUT DB sạch — không còn Dish nào tham chiếu đến ingredient đã bị xóa
✅
Bước G Verify toàn vẹn DB

Chạy query kiểm tra đảm bảo không còn FK orphan nào sót lại trong hệ thống.

-- Kiểm tra orphan: dish_ingredient trỏ tới ingredient không tồn tại SELECT di.dish_id, di.ingredient_id FROM dish_ingredient di LEFT JOIN ingredient i USING(ingredient_id) WHERE i.ingredient_id IS NULL; -- Kết quả mong đợi: 0 rows
OUTPUT 0 rows → DB toàn vẹn, pipeline scoring hoạt động chính xác
Tóm tắt luồng dữ liệu
DB Ingredient → AI Context → dishes_to_verify.txt → AI Mapping → DELETE LIST → ✓ Clean DB
